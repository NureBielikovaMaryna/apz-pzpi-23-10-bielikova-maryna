Тема роботи: Програмна система для моніторингу громадського дорожнього руху 


Мета роботи:
На цій лабораторній роботі потрібно розробити front-end частину програмної системи.


1. Опис прийнятих інженерних рішень
У ході розробки було прийнято наступні ключові рішення:
Архітектурний стиль: Використано розділення на Frontend (React.js) та Backend (ASP.NET Core), що дозволяє незалежно масштабувати частини системи.
Керування станом (State Management): Використано React Hooks (`useState`, `useEffect`) для синхронізації локального стану застосунку з БД.
Data Normalization: Реалізовано алгоритм фільтрації унікальних об'єктів на стороні клієнта за допомогою `Map`, що запобігає дублюванню даних в інтерфейсі при денормованій структурі відповідей API.
Безпека (RBAC): Впроваджено рольову модель доступу (Admin/Client). Доступ до критичних методів (створення, видалення) обмежений на рівні параметрів запиту та валідації на бекенді.
Інтеграція: Використано Microsoft Dev Tunnels для створення захищеного проксі-каналу, що дозволяє тестувати взаємодію компонентів без складного налаштування мережевих правил.


2. Опис взаємодії компонентів
Взаємодія побудована за принципом Client-Server Architecture:
1.  Frontend (React + Ant Design): Відповідає за відображення UI та збір даних через форми. Відправляє HTTP-запити через бібліотеку Axios.
2.  Dev Tunnel (Gateway): Виступає проміжним вузлом, який перенаправляє зовнішні HTTPS-запити на локальний порт `5152`.
3.  Backend (ASP.NET Web API): Приймає запити, виконує бізнес-логіку (сортування, фільтрація) та взаємодіє з базою даних через Entity Framework Core.
4.  Database (SQLite): Зберігає персистентні дані про користувачів, транспорт, станції та історію переглядів.

Висновок
У ході виконання лабораторної роботи №3 було спроєктовано та розроблено багаторівневу інформаційну систему для управління транспортною мережею.
Ключові результати роботи: 
Розробка Full-stack застосунку: Створено клієнтську частину на базі фреймворку React із використанням бібліотеки компонентів Ant Design, що забезпечило сучасний та адаптивний інтерфейс (UX/UI). Серверну логіку реалізовано за допомогою ASP.NET Core Web API, що забезпечило високу швидкість обробки запитів та гнучкість налаштування маршрутів.
Архітектурне проєктування: За допомогою мови UML було розроблено низку діаграм (Use Case, Component, Activity, State), що дозволило детально описати функціональні вимоги, компонентну структуру та динамічну поведінку системи. Це довело ефективність об'єктно-орієнтованого аналізу на етапі дизайну ПЗ.
Робота з даними: Реалізовано повноцінний механізм CRUD для управління автобусами, станціями та маршрутами. Особливу увагу приділено цілісності даних при використанні Entity Framework Core та SQLite, а також впроваджено функціонал експорту структурних даних у форматі JSON для створення резервних копій.
Інтеграція та безпека: Протестовано взаємодію компонентів у розподіленому середовищі за допомогою Microsoft Dev Tunnels. Впроваджено рольову модель доступу (RBAC), що розмежовує можливості звичайних користувачів та адміністраторів системи.
Практичне значення: Отримана система є масштабованою та підготовленою до інтеграції зовнішніх модулів (наприклад, IoT-датчиків для відстеження геолокації в реальному часі). Робота дозволила поглибити навички розробки розподілених веб-застосунків та засвоїти принципи взаємодії між Frontend та Backend частинами системи через RESTful API.
