Тема роботи: Програмна система для моніторингу громадського дорожнього руху 


Мета роботи:
На цій лабораторній роботі потрібно розробити мобільний програмний застосунок для будь-якої поширеної на сьогодні мобільної платформи. Мобільний застосунок повинен бути частиною системи і повинен взаємодіяти з нею. Для реалізації програмного застосунку можна використовувати технології для реалізації програмного забезпечення для мобільних платформ, вказані лектором, а саме Kotlin для Android та Swift для iOS.


1. Опис прийнятих інженерних рішень
У межах роботи було прийнято наступні технічні рішення:
Технологічний стек: Обрано фреймворк Flutter (Dart) через його кросплатформеність та високу швидкість розробки UI. Це дозволяє в майбутньому легко портувати застосунок з Android на iOS або Web.
Архітектура мережевої взаємодії: Оскільки серверна частина (ASP.NET Core) запущена локально, для зв'язку з мобільним клієнтом використано технологію Microsoft Dev Tunnels. Це забезпечує публічну HTTPS-адресу для локального порту 5152, що дозволяє мобільному пристрою отримувати дані через інтернет.


Управління станом (State Management): Використано асинхронні запити за допомогою бібліотеки http та віджет FutureBuilder для динамічного оновлення інтерфейсу при отриманні JSON-даних.
Автентифікація: Реалізовано клієнт-серверну авторизацію. Після успішного входу мобільний клієнт зберігає userId, що дозволяє персоналізувати історію переглядів.


2. Будова системи та її взаємодія
Система складається з трьох ключових ланок: 
IoT-симулятор (PowerShell): імітує датчик руху, надсилаючи POST-запити на сервер для оновлення затримок (delayMinutes). 
Back-end (C# + Entity Framework): обробляє запити, зберігає дані в SQLite та надає REST-ендпоінти. 
Мобільний застосунок (Flutter): виступає як інтерфейс користувача, що дозволяє реєструватися, переглядати розклад у реальному часі та історію запитів.
